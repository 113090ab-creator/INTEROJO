# 생산 진행 대시보드 운영 표준서

- 문서 버전: v1.0
- 작성일: 2026-05-14
- 기준 코드: `app.py`

## 1. 목적
이 문서는 생산 진행 대시보드의 데이터 처리, 매핑, 집계, 화면 표시, 운영 점검 기준을 표준화한다.  
향후 유지보수/배포 시 본 문서를 기준으로 동일한 결과를 재현해야 한다.

## 2. 적용 범위
- Streamlit 대시보드 전체 (`app.py`)
- 대상 화면:
1. 제품 부족수량 현황
2. 리드지 현황

## 3. 입력 데이터 표준

### 3.1 파일 종류
1. 재고 파일 (`ODV_WIP*` 우선 탐지)
2. 수요 파일 (`수요정보(전공정)` 우선)
3. 기준 파일 (`제품명 기준 정보`, BOM/분류/리드지 포함)

### 3.2 파일 선택 규칙
1. 재고 파일
- `ODV_WIP` 접두 파일이 있으면 최신 파일 사용
- 없으면 `ODV` + `WIP` 포함명에서 선택
- 그래도 없으면 파일 크기/이름 기반 fallback
2. 수요 파일
- 재고 파일을 제외한 나머지 중 `수요` 관련 파일 선택
- `전공정` 포함명이 있으면 우선
3. 기준 파일
- `제품명` + `기준` 포함 엑셀 중 최신 파일 1개

## 4. 업데이트 시간 표준
1. 화면의 `업데이트` 시간은 반드시 `수요정보(전공정).xlsx` 기준으로 표시한다.
2. 시간 산정 우선순위:
- Excel 문서 속성 `modified`
- 속성이 없거나 읽기 실패 시 파일 `mtime`
3. 표시 형식: `YYYY-MM-DD HH:MM:SS` (Asia/Seoul)

## 5. 캐시/성능 표준
1. 캐시 키 분리 원칙
- 데이터 키: 재고/수요/기준 파일 조합 (`build_data_refresh_key`)
- 기준 키: 기준 파일 단독 (`build_reference_refresh_key`)
2. 기준맵 번들 캐시
- BOM/RQ/분류/리드지 맵은 `load_reference_maps_bundle` 캐시 사용
- 수요/재고 파일 변경 시 기준맵 재생성을 최소화
3. 운영 원칙
- 수요 파일이 자주 바뀌는 운영 패턴에서 응답 지연이 커지면 캐시 키 분리 상태를 최우선 점검한다.

## 6. 데이터 처리 파이프라인

### 6.1 수요 데이터
1. 수요 시트는 `header=1`로 읽는다.
2. 상단 2행을 별도로 읽어 공정별 `생산 수량` 컬럼 위치를 동적으로 추출한다.
3. 날짜 파싱은 문자열/날짜객체/엑셀 시리얼을 모두 허용한다.
4. 정제 조건:
- 합계행 제외
- 사이트/품목 공백 또는 `nan` 제외
- 품목코드 prefix가 `P/Q/R`이 아닌 행 제외
- `부족수량`과 `사출 생산필요수량`이 모두 0인 행 제외
5. 집계 키: `사이트코드 + 이니셜 + 거래처 + 품목코드`
6. 집계 항목:
- 부족수량 합계
- 사출 생산필요수량 합계
- 납기일 최소값
- 사출 납기일 최소값

### 6.2 재고 데이터
1. 품목코드/창고/재고 컬럼은 컬럼명 후보 우선 매칭 후 fallback 인덱스 사용
2. 창고명은 내부 표준으로 정규화
- 사출창고
- 분리창고
- 검사접착창고
- 누수규격검사창고
3. 공정재고 합계 산식:
- `공정재고 합계 = 사출창고 + 분리창고 + 검사접착창고 + 누수규격검사창고`

## 7. R/Q 코드 매핑 표준

### 7.1 기본 규칙
1. 기본 추론: 품목코드 기준 `P -> R`, `P -> Q` 변환
2. 비P 코드 보정: 리드지 맵 우선, 없으면 분류정보 R/Q 맵 사용

### 7.2 BOM 적용 우선순위
1. BOM exact (`TO_ITEM_ID` 정확 일치) 최우선
2. 아직 유효하지 않은 R/Q 행에 BOM base fallback
3. 유효성 판단:
- R코드는 `R`로 시작
- Q코드는 `Q`로 시작

### 7.3 재고 매칭 보정
1. 재고 조회 시 `BUL1/BUL2`는 `BUL` 후보까지 확장 조회
2. 실제 재고키가 있는 공정코드 후보를 우선 채택

### 7.4 매핑 충돌 처리 원칙
1. `BOM exact > 유효성 보정(BOM base) > 기본/참조맵`
2. 충돌 시 상위 우선순위 결과를 강제 적용

## 8. 분류/시트분류 표준
1. P코드: `코드5` 직접 매핑
2. 비P코드: 동일 `R코드5`를 갖는 P코드의 분류 상속
3. 결측 기본값:
- 시트분류: `기타_해외`
- 분류요약: `기타`

## 9. 핵심 산식 표준
1. 전체수요 요약(분류요약별):
- 리더 부족수량 = 안전 제외 부족수량
- 안전재고 부족수량 = 안전 포함 부족수량 - 안전 제외 부족수량
- 총수요 = 리더 부족수량 + 안전재고 부족수량
2. RQ 그룹 산식:
- 사출 부족수량 = `max(사출 생산필요수량 합계 - 사출창고 합계, 0)`

## 10. 화면/필터 표준

### 10.1 상위 탭
1. 제품 부족수량 현황
2. 리드지 현황

### 10.2 제품 부족수량 하위 탭
1. 생산 현황
2. 사출 생산 현황
3. 분리 생산 현황
4. 사출/분리 공용 품목 생산 현황

### 10.3 필터 규칙
1. 사이트코드 pills (수량 합계 표시)
2. 통합검색 (콤마 `,` 기준 OR 검색)
3. 시트분류 pills
4. 분류요약 pills
5. 옵션:
- 공정재고만
- 안전 이니셜 제외
- 동일 RQ그룹만 (`R5/Q5`, `P5` 2종류 이상)

### 10.4 표/다운로드
1. 각 현황 탭에 엑셀 다운로드 버튼을 제공한다.
2. 표 컬럼 폭은 글자수 기준 `고정 px 폭`으로 설정한다.
3. 폭 계산 기준:
- 숫자형 컬럼: 더 좁은 폭 우선(요약 수치 가독성 + 화면 수용성)
- 코드형 컬럼(R/Q/P/생산/리드지): 중간 고정폭
- 긴 텍스트 컬럼(제품명/리드지명): 상대적으로 넓은 고정폭
4. 폭 산정 시 헤더 길이와 값 길이의 `p90 길이`를 함께 사용한다.
5. 표 인덱스 컬럼은 기본 숨김(`hide_index=True`)으로 가로 공간을 확보한다.
6. 긴 헤더는 축약 라벨을 사용해 폭을 절감한다.
- 예: `누수규격검사 창고 -> 누수규격`, `검사접착창고 -> 검사접착`, `공정재고 합계 -> 공정재고`
7. 숫자 표시는 천단위 구분기호 형식을 적용한다.

## 11. 리드지 현황 표준
1. 기준 시트:
- 리드지정보
- 리드지재고
2. 집계 기준:
- 대상은 P코드 부족수요
- `P코드5 -> B1코드` 매핑 후 리드지 코드 단위 집계
3. 지표:
- 생산필요수량 합계
- 최소납기일
- 창고별 재고 (합계가 0인 창고는 미표시)

## 12. 예외/오류 처리 표준
1. 필수 파일 부족 시 로딩 실패 메시지로 중단한다.
2. 주요 컬럼 미탐지 시 fallback 컬럼 후보를 순차 적용한다.
3. 날짜 파싱 실패 값은 `-`로 표시한다.
4. 재고/매핑 결측은 0 또는 기본 텍스트로 안전 대체한다.

## 13. 변경관리 표준 (운영 체크리스트)
배포 전 아래 항목을 반드시 확인한다.

1. 입력 파일 확인
- 수요/재고/기준 파일이 최신인지
- 파일명 규칙이 탐지 규칙과 충돌하지 않는지
2. 매핑 확인
- 샘플 코드 5건 이상에서 R/Q 결과가 기대값과 일치하는지
- BOM exact 우선 적용 케이스가 정상인지
3. 산식 확인
- 공정재고 합계/사출 부족수량/RQ 그룹 집계가 수동 계산과 일치하는지
4. 화면 확인
- 업데이트 시간이 수요정보(전공정) 기준인지
- 각 탭 다운로드 버튼 정상 동작 여부
5. 성능 확인
- 첫 로딩, 재로딩, 수요 파일 교체 후 재로딩 시간을 기록
- 기준맵 캐시 재사용 여부 확인
